Case study: AIVA Lab Test results

Simplified user-flow leads to higher user retention

Company

Y Soft

Role

UX designer

Timeline

2 weeks

Tools

Figma

Current UI introduction

Summary

My role

During my internship at Y Soft, I got to do a heuristic analysis of an interface AIVA Lab. Not having worked with the interface before, I had a fresh point of view and got to redesign a part of the interface- Test results.

Problem

The AIVA Lab interface used to serve developers, but now new users without technical background are working with it. This was the main reason to optimize the current interface- to serve better to non-tech people.
Users know that their test failed, but don’t know the reason. They really have to put in the work to find out what went wrong. The data they are seeking is buried beneath unimportant data.

Conclusion

The reason of failed robot’s flow should be quickly available to the user. The current interface serves unimportant test data first, important second. In order to keep the user informed about the system status, information architecture requires an improvement.

Decision

In order to immediately provide a user with all the important data, I designed a bento grid consisting of tiles summarizing important and frequently used testing data. The solution keeps user’s cognitive load on low and follows Nielsen Norman’s usability heuristics.

Heuristic analysis

Usability heuristic Problem Sollution
Visibility of system status When flow has failed, the user doesn't know why. They have to dive deep to find out. Add concrete error data and when it occured.
Aesthetic and minimalist design User first sees content, that doesn't interest them. They have to click through the important data. Change layout and prioritize data based on importance.
The bar bellow containing CTA buttons is distracting to the user, especially when it stays behind while scrolling. Change position of buttons.
Help Users Recognize, Diagnose, and Recover from Errors Error messages in logs are confusing and the user doesn't know what they mean. Change language of error messages.
Recognition rather than recall When the user wants to play the exact time an error occurred on video, they have to remember what time the error occurred then go to video and find exact time slot. Logs should be available at the same time as video.
Match between system and real world A filter icon that hides a logs filter. The icon suggests settings rather than a filter. Change icon of filtering logs.
User control and freedom It isn't clear whether the close button will close only Test results or also whole Reports. Change close buttoon to go-back button instead. The user goes babck to Reports.
Consistency and standards There is information above the video that looks like buttons but isn't clickable. Hide unnecessary video information about the recording.

Bento grid sketch

Low fidelity wireframe

I changed the layout of the Test results page. A bento grid solves the issue of immediate availability of data. This way the user sees everything at once.

Fullfiling user needs and accessibility

Iterative design

Based on the low-fi wireframe, I started designing a high fidelity prototype. The first prototype being a minimum viable product. After recieving feedback from UX-designer friends and checking accessibility of elements, I continued designing until the final design fulfilled all the user and accessibility needs.
